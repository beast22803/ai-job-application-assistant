FIELD-BY-FIELD EXTRACTION RULES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

name
  → Full name as written. Do not split into first/last.
  → If not found: return "".

email
  → Extract the email address verbatim.
  → If not found: return "".

phone
  → Extract the phone number verbatim including country code if present.
  → If not found: return "".

location
  → City and/or country as written. If both present: "City, Country".
  → If not found: return "".

linkedin
  → Full LinkedIn URL or just the profile handle — as written. Look for URLs containing "linkedin.com".
  → If not found: return "".

portfolio
  → GitHub, personal website, or portfolio URL — as written.
  → If not found: return "".

summary
  → If the resume has a summary/profile/objective section, extract it verbatim.
  → Trim whitespace but do not rephrase.
  → If not present: return "".

skills
  → Extract only technical and professional skills EXPLICITLY listed in the resume (e.g., in a "Skills" section, or explicitly named in a sentence like "Developed using React").
  → NEVER infer skills from job titles or company names (e.g., do not add "React" just because someone worked at a React agency if they don't mention using it).
  → Each item must be a specific technology, language, framework, tool, or professional methodology.
  → Remove vague entries: "hard-working", "fast learner", "communication" — these are not hard skills.
  → Deduplicate strictly: if "JavaScript" and "JS" both appear, keep "JavaScript".
  → If not found: return [].

experience (array of objects)
  → Extract every role listed in the resume — do not skip any.
  → CRITICAL: You must extract the roles in the exact order they appear, starting from the very first role listed. Do NOT skip the first role listed (the topmost or most recent role in the experience section). Every internship, research position, and full-time role must be parsed.
  → Double-check that no experience entries are duplicated and no entries are skipped.
  → CRITICAL: If the candidate held multiple distinct roles at the same company (e.g., "Full Stack Developer" and "Full Stack Developer Intern" at "Fresh Prints"), do NOT merge or skip them. You MUST extract each role as a separate entry in the experience array.
  → CRITICAL: You must NOT skip or merge work experiences with projects. Even if a work experience mentions a project that is also listed in the Projects section, you MUST still extract the work experience as a separate entry in the experience array.
  → CRITICAL: Use the EXACT keys specified: "title", "company", "start", "end", "description". Do NOT rename them to "workExperience", "dates", or anything else.
  → title: exact job title as written.
  → company: exact company name as written.
  → start: ONLY the start date of the role (e.g. from "April 2023 - December 2023", extract "April 2023"). If missing: return "".
  → end: ONLY the end date of the role (e.g. from "April 2023 - December 2023", extract "December 2023"). If current/ongoing: return "Present". If missing: return "".
  → description: concise summary (2-4 sentences) of the role's responsibilities and achievements using the resume's own language. This must be a single string, NOT an array of strings.
  → CRITICAL: Preserve all numerical metrics, percentages, dollar amounts, and concrete achievements exactly as written. Do not invent or round metrics.
  → If no experience listed: return [].

projects (array of objects)
  → Extract only projects explicitly listed in the resume.
  → name: project name as written.
  → technologies: list of technologies explicitly mentioned for this project.
  → description: 1-2 sentence summary using the resume's own language. Preserve metrics. No inventions.
  → If no projects listed: return [].

education (array of objects)
  → Extract all education entries.
  → degree: full degree name as written (e.g. "B.E. Computer Science & Engineering").
  → institution: university or college name as written.
  → year: graduation year or date range as written. If not found: return "".
  → If no education listed: return [].

certifications
  → List all certifications explicitly named.
  → Format: "Certification Name — Issuing Body" if both are present.
  → If none: return [].

languages
  → List all spoken/written languages explicitly mentioned.
  → If none: return [].
